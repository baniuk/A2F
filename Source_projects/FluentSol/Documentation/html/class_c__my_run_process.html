pobiera parametry z klasy na zewnątrz - wersja nowsza

Funkcja pobiera parametry strumienia z klasy C_Parameter_Container. Wersja nowsza - obsługa wielu wejsć w modelu. Dane odczytane z fluenta są trzymane w odzielnych obiektach SR i SUM i stąd są odrazu przepisywane do SAFEARRAY. Ta funkcja nie korzysta ze skojarzonych z całą klasą parametrów wejściowych C_myRunProcess::Parameters

Parametry:
[out] temperature temepratura
[out] fraction udział
[out] flow przepływ
[out] pressure cisnienie
[in] compNames nazwa związku - pobrana z ASPENA

Lekko zmieniona wersja - działa przy założeniu że dla kilku komponentów wszystkie struktury wyjściowe będą miały rozmiar ilości komponentów i parametry strumieni odpowiednio wpisane. Na przykład pomimo ze jest jedna temperatura do T bedzie miało rozmiar [2] i dwa razy wpisaną tą samą temperaturę zgodnie z kolejnością faz w compNames.

Zwracane wartości:
FAIL Jeśli Jakaś funkcja podrzędna zwóci FAIL (parametr nie znaleziony - raczej wszystko możliwe na etapie dewelopingu niż działania programu) lub też nie znaleziono kombinacji właściwej w ValidateParams. W tym ostanim przypadku niektóre parametry mogą mieć nieokreslone wartości
OK Jeśli sukces

Błąd:
Problem dla większej ilości wyjść

BOOL C_myRunProcess::ImportData ( void   ) 

Wczytuje dane z Fluenta.

Wczytuje dane z Fluenta. Wczytuje wszystkie wielkosci ze struktury C_Params->bEXPORTS i z nazwy pliku fulloutname Wyniki lądują w tablicach SR oraz SUM, Te tablice są klasami C_Parameter_Container, ale nie koniecznie musza przechowywać wszystkie wartości - tylko te importowane. Jeśli coś nie jest importowane to klasa ta zwraca FAIL przy próbie odczytu danego parametru

Zwracane wartości:
FAIL Jeśli błąd - nie można otworzyć pliku lub zaimportować wymaganych danych lub użytkownik podał w exports złe słowo kluczowe określające typ parametru
OK Jeśli sukces

Błąd:
poniższy fragment moze nie dzialac gdy zmieni się ilosc parametrów odczytywanych - przydatne tylko gdy zmienia sie ich konfiguracja 
 SUM.ClearAll();
 SR.ClearAll();
Błąd:
nie koniecznie to musi działać dla kilku wyjść ze wzgledu na różną ilość elementów powierzchni 
  ile_danych = ile; // ta wartosc bedzie taka sama dla wszystkich wielkosci, potrzzeban bo niektóre są średnią a niektóre sumą

domyślnie używam metod z klasy podrzędnej C_Parameter_Container bo tylko jedno wyjście 

    SUM.AssignParamToComp(b,par,SUMtmp);
    SR.AssignParamToComp(b,par,SRtmp);
